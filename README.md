# Room Booking Service (Avito Trainee)

Бэкенд-сервис для управления бронированием переговорных комнат.

## Стек технологий
* **Язык:** Go
* **Фреймворк:** Gin Gonic
* **Аутентификация:** JWT
* **Контейнеризация:** Docker, Docker Compose
* **Хранилище:** In-Memory (хранение данных в оперативной памяти)

## Реализованные возможности
В соответствии со спецификацией API реализованы следующие блоки:

### 1. Авторизация (Auth)
* **`POST /dummyLogin`** — эндпоинт для получения тестового JWT токена (роли `admin` или `user`).

### 2. Переговорки (Rooms)
* **`GET /rooms/list`** — Получение списка всех доступных комнат.
* **`POST /rooms/create`** — Создание новой комнаты (только для роли **admin**).

### 3. Расписания и слоты (Schedules & Slots)
* **`POST /rooms/{roomId}/schedule/create`** — Установка расписания для комнаты (только для **admin**). Система автоматически разбивает рабочий день на **30-минутные слоты**.
* **`GET /rooms/{roomId}/slots/list`** — Получение списка свободных слотов на конкретную дату. Сюда не попадают слоты, которые уже заняты активной бронью.

### 4. Бронирования (Bookings)
* **`POST /bookings/create`** — Создание брони на слот (только для **user**). Реализована проверка: нельзя забронировать слот в прошлом или тот, что уже занят.
* **`GET /bookings/list`** — Полный список всех броней с поддержкой **пагинации** (только для **admin**).
* **`GET /bookings/my`** — Список будущих бронирований текущего пользователя.
* **`POST /bookings/{bookingId}/cancel`** — Отмена бронирования. Метод идемпотентен и позволяет отменять только свои записи.

## Архитектура проекта
Код организован по модульному принципу:
* **`handlers/`** — Логика обработки HTTP-запросов, разделенная по доменам (bookings, rooms, auth).
* **`models/`** — Описание структур данных, соответствующих OpenAPI схеме.
* **`main.go`** — Инициализация сервера и роутинга.

## Запуск
Для запуска необходим Docker. Сервис автоматически поднимется на порту **8080**.

```bash
docker-compose up --build
```

**Проверка статуса:**
```bash
curl -i http://localhost:8080/_info
```
